# 自动看课助手经典桌面界面设计

## 目标

在不修改看课、浏览器控制、运行时安装和日志记录逻辑的前提下，重做 Tkinter 界面，使其呈现简洁、专业的 Windows 桌面工具风格。

## 视觉方向

- 使用经典专业工具布局，不采用网页式大卡片和满宽按钮。
- 使用灰白背景、细边框和分隔线组织信息。
- 使用低饱和蓝色作为主要强调色，避免大面积高饱和颜色。
- 控件采用紧凑尺寸，按钮高度约 28 至 30 像素。
- 保持 Microsoft YaHei UI 字体，兼顾中文显示和 Windows 原生观感。

## 界面结构

窗口维持约 760 × 500 的默认尺寸，并允许用户调整大小。

1. 顶部菜单栏
   - 文件：打开配置、退出。
   - 运行：开始、确认登录、停止。
   - 工具：安装浏览器组件、打开日志目录。
   - 帮助：使用说明或关于信息。

2. 紧凑工具栏
   - 开始按钮使用蓝色实心样式，作为唯一主要操作。
   - 确认登录、停止、配置和安装组件采用次级按钮。
   - 按钮继续遵循当前启用和禁用状态逻辑。

3. 运行概览
   - 展示当前状态、浏览器组件状态和当前任务。
   - 使用标准分组面板，不使用圆角卡片。
   - 状态文字与状态颜色同步变化。

4. 提示与人工处理横幅
   - 普通使用提示使用淡黄色提示条。
   - 等待答题或人工处理时显示淡红色警示条。
   - 警示状态结束后自动隐藏，不持续占用界面空间。

5. 可折叠日志区
   - 默认折叠，仅显示日志标题和最近一条活动。
   - 点击“显示详细日志”后展开原有日志文本框。
   - 展开状态下日志区域随窗口尺寸伸缩。
   - 日志写入、文件记录和事件队列逻辑保持不变。

6. 底部状态栏
   - 左侧显示当前状态。
   - 中间显示浏览器组件状态或当前任务摘要。
   - 右侧显示日志文件名称或日志入口。

## 配色规则

- 蓝色：开始、安装、处理中等主动操作或进行状态。
- 绿色：组件正常、已就绪、运行正常。
- 黄色：首次登录、普通说明和非阻塞提醒。
- 红色：等待答题、等待人工处理和运行失败。
- 灰色：禁用按钮、次要文字、边框和日志背景。

颜色仅用于表达层级和状态，不作为唯一的信息载体，所有状态同时保留文字说明。

## 交互约束

- 保留现有按钮命令、后台线程和事件队列接口。
- 安装组件期间禁用开始与安装按钮。
- 任务运行期间禁用开始按钮，启用停止和登录确认按钮。
- 日志折叠只控制显示，不停止日志记录。
- 关闭窗口时继续执行当前停止和日志关闭流程。

## 实现边界

- 主要修改 `app.py` 的布局、样式与纯界面辅助方法。
- 不修改 `Assistant`、Playwright 页面处理或课程选择逻辑。
- 可增加针对界面状态映射和日志折叠状态的单元测试。
- 不引入新的第三方 GUI 框架，继续使用 Python 标准库 Tkinter/ttk。

## 验收标准

- 界面明显呈现 Windows 专业工具风格，不像网页仪表盘。
- 主操作突出但按钮保持紧凑。
- 默认窗口中日志折叠，主要运行信息无需滚动即可查看。
- 日志可展开和收起，展开后仍能持续显示新日志。
- 已就绪、处理中、提醒和错误状态具有一致且清晰的颜色反馈。
- 原有开始、确认登录、停止、配置、安装组件和答题提醒行为全部保留。
