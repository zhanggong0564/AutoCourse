# 自动看课助手 — 设计文档

- 日期：2026-06-12
- 状态：已通过设计评审，待编写实现计划
- 目标平台：Windows，本机开发与运行（打包成 exe 为后续事项）
- 目标网站：https://www.zjzx.ah.cn/personcenter?type=train

## 1. 背景与目标

帮助用户在该继续教育平台上自动观看"未学习"的培训课程，免去全程盯屏的麻烦。
软件提供图形界面，打开浏览器后由用户手动登录一次（之后复用会话），随后自动遍历未完成课程、按网站正常速度播放视频、自动点掉挂机检测弹窗、自动进入下一节，直至全部看完。

## 3. 技术路线

采用"A 打底 + C 调试，必要时 B 补真实结构"：

- **A 通用关键字扫描**：页面操作不依赖网站具体结构，靠配置中的关键字（"未学习/继续学习/已完成"等）与 `<video>` 标签工作。先行可写，改版不易坏。
- **C 调试模式迭代**：界面提供"调试"开关，开启后把页面上看到的候选元素、视频、进度打到日志并可导出当前页 HTML，用于对着真实网站收敛选择器。
- **B 真实结构补充**：当通用关键字不足以定位时，用调试模式拿到的真实 HTML 写精确选择器。

## 4. 平台交互前提（来自用户确认）

- 登录方式：软件打开浏览器，用户首次手动登录（含可能的验证码/短信），会话持久化复用；软件不碰密码。
- 看课过程中会出现：
  - **挂机检测弹窗**（"你还在吗"之类，需点确认）→ 软件自动点掉
  - **随堂/课后答题**（多为选择题）→ 软件暂停并提醒，用户手动答题
- 不存在：人脸识别、切到后台暂停计时。

## 5. 模块划分

| 文件 | 职责 |
|---|---|
| `app.py` | Tkinter 图形界面 + 工作线程管理；开始/停止、状态、日志框、调试开关、打开配置；通过线程安全队列回传日志/状态 |
| `assistant.py` | 总调度循环；串联"找未学课程 → 打开 → 观看 → 处理答题 → 下一门"；不直接操作 DOM |
| `browser.py` | 浏览器生命周期；启动持久化 Chromium（会话存 `.browser-profile`）、检测已登录、检测登录失效 |
| `page_actions.py` | 全部页面操作（调试主战场）：`find_unwatched_course`、`play_videos`、`dismiss_idle_popup`、`is_quiz`、`is_completed`、`go_next_section`、`read_progress`；由 `config.json` 驱动 |
| `notifier.py` | 答题提醒：提示音 + 界面横幅 + 浏览器窗口提到最前 |
| `config.json` | start_url、关键字列表、选择器、轮询间隔、调试开关 |

`page_actions.py` 是路线 A/B/C 的交汇点。

## 6. 数据流（一次完整运行）

1. 界面点"开始" → 工作线程启动 `assistant.run()`
2. `browser` 启动持久化浏览器，打开 `start_url`
3. 检测登录：首次手动登录，之后自动复用会话
4. 主循环：
   - `page_actions` 在列表页找一个"未学习"课程
   - 找不到 → 全部看完，停止并提示
   - 打开课程（处理新标签页）
   - 观看循环：
     - 播放页面内的 `<video>`
     - 发现挂机弹窗 → 自动点"确定/继续"
     - 读进度写日志
     - 发现答题 → `notifier` 提醒 + 暂停，轮询直到题目消失再继续
     - 本节完成 → 进下一节 / 整门完成 → 回列表
   - 回到主循环找下一门
5. 全程日志/状态经线程安全队列回传界面，界面不卡死

## 7. 答题与挂机弹窗处理

- **挂机弹窗**：`config.json` 维护"确认按钮文字"清单，检测到自动点掉，无需用户出现。
- **答题**：检测到题目 → 提示音 + 界面横幅"需要你答题" + 浏览器提到最前 → 状态"等待答题" → 循环检查直到题目消失（用户答完离开）→ 自动继续。软件绝不替用户选答案。

## 8. 出错处理

- 网络超时/元素找不到：捕获 + 日志 + 退避重试，不崩溃，循环继续。
- 卡住不动（N 分钟无进度）：日志告警 + 自动刷新页面重试。
- 登录失效（被踢回登录页）：暂停 + 提醒重新登录，登录后自动继续。
- 用户手动关闭浏览器：检测到后优雅停止。
- 登录态异常：文档说明删 `.browser-profile` 重登。

## 9. 调试模式（路线 C）

界面"调试"勾选框。开启后 `page_actions` 把看到的候选按钮、视频、进度打到日志，并可把当前页 HTML 存成文件，用于对着真实网站逐个调准选择器（必要时顺带完成路线 B）。

## 10. 测试策略

- 纯逻辑部分（关键字匹配、进度百分比解析、答题/完成的文字判定）→ 样例字符串单元测试，按 TDD 编写。
- DOM 交互部分 → 调试模式 + 真实网站手动验证。

## 11. 运行与打包

- 当前：本机 Python 环境运行（安装依赖后启动 `app.py`）。
- 后续：打包成单个 .exe（用系统 Chrome/Edge 或自带 Chromium 的取舍留待打包阶段决定）。
