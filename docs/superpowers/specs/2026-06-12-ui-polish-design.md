# 自动看课助手界面打磨优化设计

## 目标

在已落地的经典桌面工具界面基础上，进一步提升清晰度、视觉细节、日志可读性和提醒可达性。不修改看课、浏览器控制、运行时安装和日志文件记录逻辑。

## 范围

主要修改 `app.py`，新增纯界面辅助函数并配单元测试。不动 `Assistant`、Playwright 页面处理、`runtime_installer` 与 `app_logging` 的现有行为。

## 一、高分屏清晰度

- 程序启动时（创建 `Tk` 之前）通过 `ctypes` 启用 Windows DPI 感知（System DPI Aware）。调用包裹在 try/except 中，失败时静默降级，不影响旧系统或非 Windows 平台。
- 读取实际缩放因子，按比例调整默认窗口尺寸（基准 780×500）与最小尺寸，必要时同步 `tk scaling`，保证 125% / 150% 缩放下文字锐利、布局不挤。

## 二、视觉细节打磨

- 程序图标：以 base64 内嵌 PNG 数据（纯标准库生成，不新增外部资源文件），通过 `iconphoto` 同时设置标题栏与任务栏图标，替换默认 Tk 图标。
- 窗口启动时按当前屏幕尺寸居中显示。
- 统一各分组面板与工具栏的内外边距。
- 提示条与警示横幅改为「左侧 3px 色条 + 浅色底」的经典样式：提示条为黄色系，警示横幅为红色系且字体加粗、字号略大。
- 工具栏按钮图标保留 ▶ ■ 等渲染稳定字符，间距统一；渲染不稳定的字符可替换或去除。

## 三、日志区体验

- 新增纯函数 `log_line_kind(message)` 对日志行分级：
  - 含「失败 / 错误 / 异常」→ `error`，红色；
  - 含「等待 / 暂停 / 请 」类提醒 → `warning`，琥珀色；
  - 其余 → `info`，深灰（现状）。
- 日志文本框按分级用 Text tag 着色；颜色仅辅助，文字内容不变。
- 日志上限 1000 行：超出时自动从头部删除多余行，防止长时间运行内存增长。`app.log` 文件记录不受影响。
- 日志文本框右键菜单：复制选中内容、全选、清空日志（仅清界面显示，不影响 `app.log`）。
- 展开/收起状态保持现有会话内记忆，不做跨启动持久化，不新增状态文件。

## 四、提醒强化

- 状态进入「等待答题」或「等待人工处理」时，通过 `ctypes` 调用 `FlashWindowEx` 使任务栏图标持续闪烁，直至窗口重新获得焦点；与既有 `notifier.play_alert()` 提示音并存。
- 不把窗口自动弹到前台，不设置置顶。
- 调用包裹在 try/except 中，非 Windows 平台静默降级。

## 交互约束

- 保留现有按钮命令、后台线程与事件队列接口。
- 状态颜色映射、横幅显隐逻辑、按钮启用/禁用逻辑维持现有行为。
- 清空日志只影响界面 Text 控件，不中断日志写入。

## 测试

- `log_line_kind` 的分级判定（错误、提醒、普通三类）。
- 日志行数裁剪的纯逻辑（如有独立辅助函数）。
- 现有 `runtime_button_state`、`status_presentation`、`log_toggle_text` 测试保持通过。
- DPI、图标、闪烁等依赖 Windows API 的部分以降级安全为准，人工验收。

## 验收标准

- 125% / 150% 缩放屏幕上界面文字清晰不发虚。
- 标题栏与任务栏显示自定义程序图标，窗口启动居中。
- 日志中错误行红色、提醒行琥珀色，右键可复制/全选/清空，长时间运行日志不超过 1000 行。
- 等待答题时任务栏闪烁直至用户点击窗口，窗口不强行抢焦点。
- 原有开始、确认登录、停止、配置、安装组件与答题提醒行为全部保留。
